# Geographic Search Feature

## Goal

A search box on the interactive map that lets users find and fly to clusters or cluster groups by name, city, or county.

## Approach

**Local data search ŌĆö no new dependencies.** The dataset is small (~75 clusters + 5 groups), so simple case-insensitive `includes()` filtering is sufficient. No fuzzy search library needed. Bbox calculation uses a small inline helper instead of `@turf/bbox`.

## What's searchable

Each cluster is indexed by multiple terms:

| Source | Example |
|--------|---------|
| Cluster code | `IN-05` |
| Cluster name | `Hollow Reed` |
| Notable cities (`clusterCities.ts`) | `Bloomington`, `Columbus`, `Seymour` |
| Counties (`clusterCounties.ts`) | `Monroe`, `Brown`, `Bartholomew` |

Groups are indexed by code (`INDY`), display name (`Indianapolis`), and the word `group`.

### Match context

When a search matches on a term that isn't visible in the result (e.g. a cluster name or county), a blue context tag appears next to the cluster code explaining why it matched:

- Searching "hollow" ŌåÆ `CLUSTER IN-05` with tag `Name: Hollow Reed`
- Searching "wayne" ŌåÆ clusters containing Wayne County, each tagged `Wayne County`
- Searching "indianapolis" ŌåÆ no tag needed (already visible in label/sublabel)

Context is computed at index-build time by comparing each key's display label against the visible result text.

## Search logic

On each keystroke (debounced 150ms):

1. Find entries where any key starts with the query (prefix matches)
2. Find entries where any key contains the query (substring matches)
3. Prefix matches first, then substring, each sorted alphabetically
4. Cap at 8 results

## Fly-to behavior

On result selection:

- **Cluster:** `fitBounds()` on the cluster's bbox
- **Group:** `fitBounds()` on the combined bbox of all member clusters
- **Padding:** 160px on desktop (cluster fills ~40-50% of viewport), 80px on mobile

## Files

| File | Purpose |
|------|---------|
| `src/components/FloatingSearch.tsx` | Search input + dropdown UI |
| `src/lib/useSearchIndex.ts` | Builds search index from GeoJSON features, returns `search()` function |
| `src/lib/featureBbox.ts` | Inline bbox helper (~10 lines, replaces `@turf/bbox`) |
| `src/data/clusterCities.ts` | Notable cities per cluster (hand-curated) |
| `src/data/clusterCounties.ts` | Counties per cluster (generated by `MapCountiesToClusters`) |

Wired into `RegionMap` ŌĆö renders inside `MapProvider` (needs `useMap()` for fly-to), gated on `!printMode`.

## UI

- **Position:** Fixed top-left (`top-4 left-4`), z-index 60
- **Width:** Full width with margins on mobile (`left-4 right-4`), fixed `w-72` on desktop (`sm:w-72`)
- **Styling:** White background, `shadow-lg`, rounded corners ŌĆö matches other floating controls
- **Keyboard:** Arrow keys to navigate, Enter to select, Escape to close
- **Result format:** Type badge (`CLUSTER`/`GROUP`), label, optional match context tag, sublabel (cities or group code)

## Bundle size impact

~0. Both GeoJSON files are already in the shared JS bundle (imported by `ClusterLayers` and `CountyBoundaries`). The city/county data files and search component add a few KB at most. No new npm dependencies.
